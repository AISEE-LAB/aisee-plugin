# Brief Batch Index Template

当单个 OpenSpec change 内容过大，需要拆成多个 implementation handoff part 时，先生成本索引。

规则：

- `brief-index.md` 是分批调度索引，不是规范事实源。
- 不复制 proposal、specs、contracts、source-map 或 tasks 正文。
- 只记录批次边界、依赖顺序、读取入口、允许路径、验证入口和回写位置。
- 每个 part 仍使用 `brief-template.md`。

## Global Guardrails

- Do not expand current change scope.
- Do not copy artifact bodies into brief parts.
- Do not create a parallel durable task list.
- If a batch discovers artifact conflict, stop and update the current change artifact first.

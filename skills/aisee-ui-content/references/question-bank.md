# aisee:ui-content — 问题库

在 Phase 1 输入门禁和后续澄清时读取。只选择真正影响 UI 内容规格的问题，不要全部询问。

---

## 追问优先级

1. 平台范围
2. 场景模式与现有 UI 事实来源
3. 用户角色与权限差异
4. 核心任务路径
5. 页面承载方式
6. 表单 / 列表 / 状态反馈细节
7. 多端能力差异

每轮最多 3 个问题。超过 3 轮仍不明确时，把剩余问题写入 Open Questions。

---

## 平台范围

- 这个功能要覆盖哪些端：PC Web / H5 / App / 微信小程序？
- 如果是多端，哪些端必须首发，哪些可以后续补？
- 不同端的用户角色是否相同？

---

## 页面清单与入口

- 用户从哪里进入这个功能：导航、列表行操作、通知、分享链接、待办、扫码，还是其他入口？
- 完成主任务后，用户应该回到哪里：列表、详情、首页、结果页，还是停留当前页？
- 是否有必须作为独立页面承载的内容，还是可以用弹窗/抽屉/Bottom Sheet？

---

## 现有系统二开

- 这是新建系统 / 新模块，还是基于现有系统二次开发？
- 如果是二开，现有页面事实来源在哪里：历史 UI 文档、OpenSpec baseline、source-map、路由、页面代码、截图，还是产品说明？
- 本次需求会影响哪些既有页面、入口、弹窗或流程？
- 哪些既有页面只是复用，不需要重新展开完整页面规格？
- 是否存在新增页面、下线入口、替换旧流程或需要兼容旧入口的情况？
- 现有页面名称、路由或 ID 是否需要沿用，还是允许在 UI Content 中补充临时 ID？
- 有哪些现有行为无法从材料确认，需要标为 Unknown 而不是假设？

---

## 用户角色与权限

- 哪些角色可以访问该功能？
- 不同角色看到的数据范围是否不同？
- 哪些按钮、字段或操作需要按角色隐藏、置灰或只读？
- 无权限时是隐藏入口、置灰提示、跳转 403，还是展示申请权限入口？

---

## 信息架构

- 这个功能属于现有哪个模块或导航层级？
- 是否需要全局入口，例如消息中心、待办中心、个人中心、设置或帮助？
- 是否存在跨模块流程，需要单独说明流程而不是放在某个模块页面里？

---

## 表单与输入

- 哪些字段必须由用户填写，哪些字段由系统自动带出？
- 是否支持草稿、暂存、重复提交防护或未保存离开提醒？
- 字段之间是否有联动、只读、禁用或动态必填规则？
- 提交失败时用户在哪里看到错误，是否保留已填写内容？

---

## 列表、筛选与详情

- 列表必须展示哪些列，哪些列只在 PC 展示、移动端隐藏或折叠？
- 需要哪些搜索、筛选、排序、分页或批量操作？
- 详情页必须展示哪些信息，哪些内容可折叠或放入关联数据区？
- 长文本、图片、文件、金额、日期、状态枚举如何展示？

---

## 状态与反馈

- 无数据、搜索无结果、加载失败、无权限时分别给用户什么反馈？
- 提交中、提交成功、提交失败分别如何反馈？
- 异步任务（导入、导出、同步、审批）在哪里查看结果？
- 通知是否需要跳转到某个落地页？

---

## 危险操作

- 删除、撤销、批量操作、状态变更、支付、提交审批是否需要二次确认？
- 操作成功后是否可撤回？
- 部分成功时如何展示成功项、失败项和重试入口？

---

## 多端差异

### PC Web / Admin

- 是否需要高密度表格、批量操作、复杂筛选或导出？
- 是否存在多窗口、侧边详情、快捷操作等管理后台习惯能力？

### H5

- 是否运行在微信内置浏览器或普通浏览器？
- 是否需要考虑软键盘、分享、登录态丢失或浏览器返回？

### App

- 是否使用相机、相册、定位、扫码、推送、离线或弱网能力？
- App 切后台或页面退出后是否保留填写内容？

### 微信小程序

- 是否需要微信授权、手机号授权、订阅消息、分享、支付或扫码？
- 是否受页面栈、审核规则、下载导出能力限制影响？

---

## 文案

- 页面标题、主按钮、空状态、错误提示是否有业务指定文案？
- 确认弹窗是否需要明确风险说明？
- 权限不足、授权失败、网络异常时的提示语是否有合规或品牌要求？

---

## Open Question 处理

如果用户没有回答但不阻塞页面清单生成，记录为：

`[OPEN QUESTION] {问题} — 影响页面 {PAGE-xxx} — 影响 FR {FR-xxx} — 影响平台 {platform}`

只有在平台范围或主流程无法判断时才停止等待用户回答。

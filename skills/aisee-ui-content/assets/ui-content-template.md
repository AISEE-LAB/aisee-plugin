# aisee:ui-content — UI 内容规格模板

在 Phase 5 读取本文件。根据输出模式选择“标准文档模板”或“Epic 模式模板”。

---

## 质量检查清单

生成前逐项检查：

- [ ] 没有写视觉设计规范、布局稿、组件库或实现方案
- [ ] 所有页面都有页面 ID、页面目标、入口、完成后去向、关联 FR
- [ ] 所有 FR 都能追踪到页面、元素、操作或非页面型系统行为
- [ ] 平台范围已明确，未默认 PC 或移动端
- [ ] 多角色差异已覆盖入口、字段、按钮、数据范围和操作权限
- [ ] 表单字段包含必填、默认值、校验、联动、只读/禁用、提交反馈
- [ ] 危险操作包含确认、取消、成功、失败、恢复规则
- [ ] 状态反馈覆盖加载、空、无结果、错误、无权限、提交中、成功、失败
- [ ] Open Questions 只放未确认事项，不把假设写成事实
- [ ] 标准模式没有超过 8 个页面详情

## 写作规则

- 删除所有未使用的模板占位段落，不要留下 `{placeholder}`。
- 如果某节确实无内容，写“无”，不要删除关键章节。
- 页面锚点使用小写页面 ID，例如 `### PAGE-001` 可被链接为 `#page-001`。
- Epic 模式索引文档只写总览，不展开完整页面详情。
- 模块文档可独立阅读，不要求读者先读索引才能理解页面内容。
- 平台差异以“内容、入口、操作、授权、反馈”为中心，不写视觉布局。
- 埋点候选是可选章节；如果需求没有分析诉求，可以写“无”。

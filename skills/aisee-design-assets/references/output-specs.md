# 输出规格

## 目标

按平台、用途和最终展示面积选择尺寸、清晰度、安全区和导出策略。不要把同一套大尺寸要求套到所有图片上。

## 触发条件

仅在以下情况读取本文件：

- 用户要求生成参考图、素材图、图生图版本或透明素材。
- 平台、端类型、尺寸、比例、清晰度或安全区会影响结果。
- 需要生成素材清单、开发视觉 brief 或 Figma brief。

## 选择原则

- 尺寸按最终展示面积反推，不按“素材”统一给大图。
- 参考图默认不低于实际评审用途的 2x 清晰度；草图可降低。
- 移动端参考图使用真实设备比例，不随意使用正方形或 16:9。
- 使用 `gpt-image-2` CLI 时，生成尺寸应满足脚本校验；真实设备尺寸可在后处理阶段裁切或缩放。
- PC / 后台参考图优先信息清晰度和可读性。
- 长页面可以按首屏、关键区块和状态页分图生成，不强行塞进一张超长图。
- 图标和文字密集 UI 的最终资产用 SVG、组件库、Figma 或前端实现，不依赖生图精确输出。

## 推荐规格

| 场景 | 推荐尺寸 | 要求 |
|---|---:|---|
| 移动 App 参考图 | `1296x2800` / `1184x2560` | 接近真实手机比例，至少 2x 评审清晰度 |
| 小程序页面参考图 | `752x1632` / `1120x2432` | 接近 750 设计稿逻辑，必要时 3x |
| H5 首屏 | `1088x1920` | 首屏清晰；长页按区块拆分 |
| PC Web 页面 | `1440x1024` / `1920x1088` | 适合桌面评审和 Figma 参考 |
| 管理后台 / Dashboard | `1440x1024` / `1600x1008` | 信息密度清晰，不牺牲可读性 |
| 大屏可视化 | `3840x2160` | 4K 优先，文字和图表可读 |
| Hero / Banner | `1920x800` / `2048x1152` | 预留裁切安全区 |
| 通用图标 | SVG 优先；预览 PNG `128x128` / `256x256` | 优先图标库，不靠生图 |
| 自定义小图标 | `256x256` / `512x512` | 品牌或业务专属符号，最终尽量矢量化 |
| 透明 UI 小素材 | `256x256` / `512x512` | badge、贴纸、小装饰、按钮装饰 |
| 透明插画素材 | `1024x1024` 起 | 仅复杂插画、角色、营销视觉使用 |
| Hero / 背景装饰透明素材 | 按最终展示面积和画布比例 | 大面积展示时才使用 `1024+` |

## 图生图视觉变体

图生图视觉变体默认是受控修改，不是重新设计。除非用户明确要求重设风格，否则必须保持：

- 品牌元素：Logo、品牌色、品牌字体、品牌图形、吉祥物、IP 形象。
- 版式结构：导航、卡片、按钮、模块位置、留白节奏。
- 视觉风格：线条粗细、圆角、阴影、材质、插画风格、摄影风格。
- 色彩系统：主色、辅助色、背景色、强调色比例。
- 字体与文字层级：字号、字重、标题/正文/标签关系。
- 图标风格：线性/面性、stroke、端点、圆角、尺寸。
- 业务语义：字段、状态、按钮含义、关键数据。
- 不应变动区域：默认保持不变，尤其是文字和品牌标识。

按需加入最小约束：

```text
Preserve the original brand identity, layout structure, visual style, color system, typography hierarchy, and icon style. Only modify the requested area or elements. Keep unmasked areas unchanged.
```

涉及 Logo、品牌标识或真实产品图时，默认保留原像素或使用确定性贴图替换，不要求模型重画。

需要 mask、bbox、cutout、背景修补或从已有图片提取素材时，先进入 `aisee:image-object`。本文件只用于确定参考图、生成型素材和视觉变体的输出规格。

## 记录字段

索引、manifest 或 brief 中建议记录：

- 平台和用途
- 输出尺寸
- 生成模式：text-to-image / image-to-image / image-object handoff
- 参考图、StyleSpec 或源图路径
- 保留项和修改项摘要
- 风险：文字、Logo、细节漂移、透明边缘、尺寸不足等

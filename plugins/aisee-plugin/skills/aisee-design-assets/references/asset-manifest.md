# 素材清单与复核

## 素材类型

- `background`：页面背景、Hero 背景、模块背景
- `icon`：入口图标、功能图标、状态图标
- `illustration`：插画、场景图、人物/物体视觉
- `overlay`：光效、纹理、装饰层
- `layout-layer`：卡片底板、容器背景、视觉框架层
- `transparent-generated`：生成型透明 PNG，不来自既有图片对象提取
- `extracted-object`：由 `aisee:image-object` 从已有图片提取的对象、cutout、export 或图层包

## 生成前必须明确

- 素材用途
- 依赖的参考图或 StyleSpec
- 尺寸或比例
- 是否透明
- 是生成型素材，还是需要从已有图片提取
- 是否允许文字、Logo、UI 控件
- 素材边界：整张图、背景层、插画主体、装饰层、局部区域、透明主体或仅登记外部对象
- 文本策略：不生成文字、预留可编辑文字区、还是已确认要生成 raster text
- 输出目录
- 是否属于通用图标；通用图标优先记录库名和语义，不走生图

## Asset Intent Scan

生成前先识别“哪些视觉元素应该成为素材”，不要把参考图里所有东西都交给图片模型。

| 视觉元素 | 默认判断 | 处理策略 |
| --- | --- | --- |
| 页面背景、Hero 背景、渐变、纹理、抽象光效 | 生成型素材 | 由本 skill 生成，记录 style_anchor 和安全区 |
| 插画、角色、业务专属图形、营销主视觉 | 生成型素材 | 由本 skill 生成，严格锁定风格和用途 |
| Banner 底图、活动视觉底板 | 生成型素材 | 默认只生成底图和装饰，不生成可编辑文案 |
| Banner 标题、价格、日期、按钮、角标 | UI 文案或运营配置 | 前端/Figma 可编辑；只有最终投放图才考虑 raster text |
| 通用导航、操作、状态图标 | 图标库素材 | 按 `icon-library.md` 映射，不走生图 |
| Logo、品牌字标、真实商标 | 既有品牌资产 | 不生成，必须引用现有资产 |
| 产品照片、人物照片、已有图片对象 | 既有图片对象 | 交给 `aisee:image-object` 提取、去背景或修边 |
| 卡片、按钮、表单、Tab、列表项 | UI 组件 | 不生成，交给组件库/Figma/前端实现 |
| 纯装饰 shape、分割线、容器底板 | CSS/SVG 优先 | 复杂材质或氛围层才生成 |

Banner 文字策略：

- App/Web/小程序运营位：默认 `text_policy=no-text` 或 `editable-text-zone`，图片只生成背景、插画、装饰层和留白安全区。
- 社媒图、电商主图、最终投放海报：只有用户确认文案、字体风格、层级和不可编辑风险后，才允许 `text_policy=raster-text-confirmed`。
- 参考图里已有文字但下游是前端实现时，prompt 必须明确避免生成文字、Logo、水印和按钮文案。
- 必须还原带文字 banner 时，同时记录可编辑重建风险；文字准确性不能仅依赖图片模型。

建议在 manifest 中先写决策，再生成：

```markdown
| 区域 | 视觉元素 | UI/素材判断 | generation_decision | asset_boundary | text_policy | 图标策略 | 还原要求 | 风险 |
|------|----------|-------------|---------------------|----------------|-------------|----------|----------|------|
```

## 状态

- `candidate`：候选，未确认
- `usable`：可进入下游流程
- `needs-edit`：可保留但需要人工微调或继续编辑
- `rejected`：不建议使用
- `archived`：历史版本，仅用于追溯

## 复核清单

参考图：

- 平台比例是否正确
- 是否符合页面场景
- 是否有不该出现的文字、Logo、水印或 UI 控件
- 是否能作为视觉方向，不误导为最终稿

设计规范：

- 是否能追溯来源
- 是否覆盖色彩、构图、图形语言、素材需求和禁用项
- 是否足以驱动下一轮生成
- 是否过度抽象或含糊

素材：

- 是否符合用途
- 是否与参考图或 StyleSpec 一致
- 图标是否优先来自图标库，语义和风格是否一致
- 生成型透明素材是否存在背景残留、绿边、毛边或错误阴影
- 提取型素材是否有 `aisee:image-object` workspace、cutout/export 路径、mask 来源和边缘复核结论
- 是否需要重做、转交 `aisee:image-object` 或人工微调

## Manifest 字段

```markdown
| ID | 文件 | 类型 | 用途 | 来源 | generation_decision | asset_boundary | text_policy | 透明 | 尺寸 | 状态 | 备注 |
|----|------|------|------|------|---------------------|----------------|-------------|------|------|------|------|
```

图标素材可追加语义字段：

```markdown
| 语义 | 推荐图标 | 备选 | 尺寸 | 风格 | 用途 | 状态 |
|------|----------|------|------|------|------|------|
```

## 2. Directory Semantics

| Directory | Meaning | AI Rule |
|---|---|---|
| Drivers/ | Vendor libraries, CMSIS, HAL/LL, SDK, low-level third-party packages | Do not edit unless explicitly requested |
| Core/ or FunctionB/ | MCU peripheral init, main, interrupts, system clock, DMA/GPIO/USART/ADC/SPI setup, main scheduling | Edit carefully; preserve vendor generated sections |
| Function/ or FunctionE/ | Extended peripherals, algorithms, business modules, display, protocol, measurement/control logic | Preferred location for new feature modules |
| docs/ | Requirements, contracts, design, debug notes, verification records | Keep updated when contracts change |
| tools/ | Optional scripts, log parsers, flashing helpers, data generation | Create only when needed |

## 5. Guardrails

- Do not modify IDE project files unless the user explicitly allows it.
- If IDE project files are modified, back them up first.
- Do not move vendor-generated files without documenting the build impact.
- New feature code should prefer `Function/<module>/` unless the contract says otherwise.
- If the project uses `FunctionB/FunctionE`, preserve those names instead of renaming.
